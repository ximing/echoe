## 2. 产品目标

### 2.1 核心目标

1. **功能完整性**: 支持 95% 以上 Anki 核心功能
2. **数据兼容性**: 100% 兼容 .apkg 格式
3. 算法使用 FSRS 算法，可以使用 ts-fsrs 库

## 4. 用户故事 (User Stories)

### 第一阶段：核心学习引擎 (MVP)

#### US-001: 实现间隔重复算法

**描述**: 作为学习者，我需要应用使用科学的间隔重复算法，以便在最佳时间复习卡片。

**验收标准**:

- [ ] 支持fsrs
- [ ] 每个评分对应不同的间隔时间计算
- [ ] 算法参数可配置
- [ ] 算法实现通过单元测试（覆盖常见场景）
- [ ] 算法结果与 Anki 官方实现对比验证

#### US-002: 学习会话管理

**描述**: 作为学习者，我需要应用管理我的每日学习任务，显示待复习和新卡片数量。

**验收标准**:

- [ ] 主界面显示今日待学习卡片总数

- [ ] 学习会话显示当前进度（如 15/50）
- [ ] 完成学习后显示完成界面和学习统计
- [ ] 支持中途退出并保存进度
- [ ] 显示本次学习会话时长计时器
- [ ] 支持自定义每日开始时间（如凌晨4点）

#### US-003: 卡片展示与答题

**描述**: 作为学习者，我需要在移动设备上方便地查看卡片内容并标记掌握程度。

## 5. 功能需求 (Functional Requirements)

### 5.1 核心学习模块

#### FR-001: fsrs 算法实现

- 实现标准 fsrs 算法
- 支持可配置的学习参数：
  - 新卡片步长（分钟）：默认 1, 10
  - 毕业间隔（天）：默认 1
  - 简单间隔乘数：默认 1.3
  - 间隔乘数：默认 2.5
  - 最大间隔（天）：默认 36500
  - 起始易度：默认 2.5
  - 易度调整因子：默认 0.15
- 算法必须考虑：延迟复习、重新学习队列、学习队列

#### FR-002: 卡片调度系统

- 每日零点重置学习计数
- 支持时区设置
- 处理跨时区旅行后的卡片调度
- 支持提前复习（在到期日前复习）

#### FR-003: 卡片渲染引擎

- 支持 HTML 子集：div, span, p, br, b, strong, i, em, u, strike, a, img, audio, ul, ol, li
- 支持内联 CSS 样式
- 支持字段替换 `{{字段名}}`
- 支持条件渲染 `{{#字段名}}...{{/字段名}}`
- 支持填空删除 `{{c1::文本}}`, `{{c1::文本::提示}}`
- 支持 LaTeX 渲染（使用 MathJax 或 KaTeX）

#### FR-004: 媒体处理

- 图片格式：JPEG, PNG, GIF, WebP, SVG
- 音频格式：MP3, AAC, OGG, WAV
- 视频格式：MP4（可选）
- 图片自动压缩选项
- 媒体文件与卡片关联存储

### 5.2 数据管理模块

#### FR-005: Anki 数据库兼容

- 使用 Mysql 作为数据库 禁止使用外键
- 数据库结构兼容 Anki 2.1 格式：
  - `cards` 表：卡片状态、间隔、易度等
  - `notes` 表：笔记内容、字段、标签
  - `revlog` 表：复习历史记录
  - `decks` 表：牌组配置
  - `notetypes` 表：笔记类型定义
  - `templates` 表：卡片模板
  - `media` 表：媒体文件索引
  - `graves` 表：删除记录（用于同步）
  - `config` 表：应用配置
  - `col` 表：集合元数据

#### FR-006: .apkg 格式支持

- 解析 ZIP 压缩的 .apkg 文件
- 导入媒体文件到本地存储
- 处理数据库版本迁移
- 导出时生成有效的 .apkg 文件

### 5.3 用户界面模块

#### FR-008: 主界面布局

- 顶部：应用标题 + 同步按钮 + 设置入口
- 中部：牌组列表（可折叠层级）
- 底部：今日概览（总待学习数）
- 空状态：引导用户导入或创建牌组

#### FR-009: 学习界面

- 卡片展示区域（占屏幕 70%）
- 底部操作区域（显示答案/评分按钮）
- 顶部状态栏（进度、计时器、菜单）
- 支持横屏/竖屏切换

#### FR-010: 卡片浏览器

- 列表视图：卡片预览、牌组、复习日期
- 搜索功能：全文搜索、筛选器
- 排序选项：添加日期、复习日期、随机
- 多选操作：删除、移动、暂停、标记

### 5.4 高级功能

#### FR-011: 笔记类型支持

- 基础类型：基础、基础（输入答案）、基础（翻转）、基础（可选翻转）
- 填空类型：填空、填空（输入答案）
- 自定义类型支持

#### FR-012: 筛选搜索语法

- 基础搜索：关键词匹配
- 字段搜索：`字段名:关键词`
- 标签搜索：`tag:标签名`
- 牌组搜索：`deck:牌组名`
- 状态搜索：`is:new`, `is:review`, `is:learn`, `is:suspended`, `is:buried`, `is:marked`
- 日期搜索：`added:1`（今天添加）, `rated:7`（7天内评分）
- 逻辑运算：`and`, `or`, `-`（排除）

#### FR-013: 撤销/重做系统

- 学习撤销：维护最近评分操作栈，支持撤销后恢复上一张卡片
- 编辑撤销：基于文本编辑器的撤销栈实现
- 撤销栈大小：至少保留 5 步历史
- 持久化考虑：重要操作（如删除）支持撤销直到应用关闭

#### FR-014: 卡片搁置与延期

- 手动搁置：将卡片推迟到第二天学习
- 兄弟卡片搁置：当学习某张卡片时，自动搁置同一笔记生成的其他卡片
- 搁置状态存储：在 cards 表中记录 buried 状态
- 每日重置：每天首次打开应用时清除搁置标记

#### FR-015: 卡片状态管理

- 忘记操作：重置卡片为新状态，interval=0, reps=0, lapses++
- 重置选项：可选择是否重置卡片位置（due 值）
- 批量操作：卡片浏览器支持多选后批量忘记

#### FR-016: 输入答案对比

- 输入框渲染：在卡片正面显示文本输入框
- 答案对比：背面显示用户输入与正确答案的差异
- 差异算法：字符级对比，标记正确/错误/缺失/多余
- 配置选项：忽略大小写、忽略首尾空格、忽略所有空格

#### FR-017: 音频与TTS

- 自动播放：支持正面/背面/都不自动播放配置
- TTS 引擎：集成系统 TTS，支持语言选择
- 字段映射：可为不同字段指定不同语音
- 播放控制：播放/暂停/重播/进度条

#### FR-018: CSV/TSV 导入

- 文件解析：支持 UTF-8、GBK 等编码自动检测
- 分隔符：自动检测逗号、制表符、分号
- 字段映射：可视化选择列与笔记字段的对应关系
- 预览功能：显示前 5 行预览
- 错误处理：记录导入失败的行号和原因

#### FR-019: 媒体管理

- 未使用检测：扫描 media 表与实际引用，找出孤儿文件
- 批量删除：安全删除未使用的媒体文件
- 大小统计：按类型汇总媒体文件大小
- 反向查找：从媒体文件定位引用它的所有卡片

#### FR-020: 重复检测

- 相似度算法：基于 Levenshtein 距离或 n-gram
- 字段选择：可指定用于比较的字段
- 阈值设置：可调节相似度匹配阈值
- 对比视图：并排显示疑似重复卡片

#### FR-022: 小部件系统

- 小部件类型：2x1（仅总数）、4x1（各牌组）、4x2（详情）
- 刷新机制：系统触发 + 应用学习完成后主动刷新
- 数据缓存：本地存储上次查询结果
- 点击路由：根据小部件类型跳转到对应牌组学习界面

#### FR-023: 学习提醒通知

- 本地通知：使用鸿蒙系统通知服务
- 提醒时间：可设置每日提醒时间
- 智能提醒：仅在有待学习卡片时发送
- 通知内容：显示待学习卡片数量和预估学习时间

#### FR-024: 顽固卡片检测

- 失误计数：记录每张卡片的连续失误次数
- 阈值检测：默认 8 次失误触发 Leech 标记
- 自动暂停：标记为 Leech 时自动暂停卡片
- 统计展示：学习界面显示当前卡片的失误历史
- 批量处理：卡片浏览器中可筛选和处理 Leech 卡片

#### FR-025: 卡片重排序

- 位置字段：支持调整新卡片的 due 值（队列位置）
- 批量重排：选择多张卡片批量设置新位置
- 随机分布：将卡片随机分散到未来 N 天
- 牌组内重排：仅影响当前牌组的卡片顺序

#### FR-026: 自定义学习

- 额外新卡片：临时增加今日新卡片学习量
- 提前复习：允许复习未到期的卡片（可选是否更新调度）
- 预览模式：浏览卡片但不记录复习
- 范围选择：可限制在特定牌组或筛选条件

#### FR-027: 笔记类型管理

- 字段管理：添加/删除/重命名字段，设置字段属性
- 模板管理：添加/删除/编辑卡片模板（正面/背面/CSS）
- 模板预览：编辑时实时预览渲染效果
- 克隆功能：基于现有类型创建新类型
- 类型删除：仅允许删除未使用的笔记类型

#### FR-028: 标签管理系统

- 标签列表：显示所有标签及使用次数
- 重命名：批量更新所有引用的卡片
- 合并：将多个标签合并为一个
- 清理：删除系统中未使用的标签
- 颜色编码：支持为标签设置颜色

#### FR-029: 导入更新模式

- 更新匹配：基于字段匹配更新已有笔记
- 匹配策略：可选择匹配字段（如按第一个字段匹配）
- 合并选项：选择保留本地或导入的数据
- 冲突报告：显示哪些笔记被更新、添加、跳过

#### FR-030: 学习计时统计

- 会话计时：记录每次学习的时长
- 每日汇总：统计每日总学习时长
- 牌组细分：按牌组统计学习时间
- 显示位置：学习界面显示当前会话时长

#### FR-031: 跨天学习处理

- 队列固化：学习会话开始后锁定当日卡片队列
- 午夜处理：跨午夜后不影响当前学习会话
- 日界设置：支持自定义每日开始时间（如凌晨4点）

#### FR-032: 筛选牌组预览

- 预览功能：创建筛选牌组前显示匹配结果数量
- 实时更新：修改搜索条件时实时更新预览

- 快速应用：一键应用预设到新牌组
- 默认预设：设置新建牌组的默认配置

| 填空      | Cloze             | 一种卡片类型，隐藏句子中的部分文字           |
| 易度      | Ease              | 表示卡片难度的数值，影响间隔计算             |
| 间隔      | Interval          | 两次复习之间的时间间隔                       |
| 搁置      | Bury              | 将卡片推迟到明天学习                         |
| 暂停      | Suspend           | 将卡片从学习队列中移除，直到手动恢复         |

| 失误      | Lapse             | 复习阶段卡片点击"重来"的次数                 |
| 失误次数  | Lapse Count       | 卡片历史上总失误次数                         |
