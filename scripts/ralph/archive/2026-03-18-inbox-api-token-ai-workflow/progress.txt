## Codebase Patterns
- Database schemas use Drizzle ORM with VARCHAR(191) for primary/foreign keys (MySQL utf8mb4 index limit)
- Timestamps use `timestamp('field_name', { mode: 'date', fsp: 3 })` for millisecond precision
- All schemas must be exported from `apps/server/src/db/schema/index.ts`
- Soft deletes use `deleted_at` timestamp field with nullable type
- Business IDs follow pattern: type prefix + nanoid (e.g., 'at' for api_token)
- Always run `pnpm build` before generating migrations (drizzle-kit reads from dist/)
- Index naming convention: `{field}_idx` for single-field indexes
