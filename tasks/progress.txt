## Codebase Patterns
- Database schemas: Use int auto-increment for id, varchar(191) for uid/name (MySQL utf8mb4 index limit)
- Tenant-scoped entities: Always add unique constraint on (uid, name) for multi-tenant name fields
- Timestamps: Use `timestamp('created_at', { mode: 'date', fsp: 3 })` for millisecond precision
- Schema exports: Must export from schema/index.ts using .js extension (drizzle-kit reads compiled JS)
- Migrations: Always build server first (`pnpm --filter @echoe/server build`) before `migrate:generate`
- DTOs: Create separate DTO file for related entities, export from index.ts with .js extension
- DTO list responses: Include items array and total count
- Services: Use @Service() decorator from typedi, ensure uid scoping in all queries
- Auto-seeding: Check count on first list() call, seed defaults if count = 0
- Idempotent create: Check if entity exists before inserting, return existing if found
- Delete pattern: Fetch entity first, update related records to null, then delete entity
- Controllers: Use @JsonController('/api/v1/resource'), inject services via constructor, must export from controllers/index.ts
- Controller auth: Use @CurrentUser() decorator, always check if userDto?.uid exists before processing
- Controller validation: Trim whitespace from user input, validate required fields, return proper error codes
- Controller delete: Use @Delete('/:id') with @Param('id'), parse and validate ID, return success message
- Controller responses: Use ResponseUtil.success() and ResponseUtil.error() for consistency
- Controller tests: Mock services, test happy path + error cases, use ! operator for result.data after null check
