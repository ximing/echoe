# Echoe ID Refactor (Big Bang) - Progress

## Codebase Patterns
- Use business ID columns (`*_id`) for all domain relations; keep numeric `id` only as internal auto-increment primary key.
- Use `generateTypeId(OBJECT_TYPE.XXX)` for all new Echoe domain IDs.
- Preserve uid-scoped query isolation in all backend read/write paths.
- Treat migration as one-shot Big Bang change set with strict quality gates.
