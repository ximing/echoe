# Echoe Multi-user Refactor - Progress

## Codebase Patterns
- **Drizzle schema unique constraints**: Use `unique('constraint_name').on(table.col1, table.col2)` for multi-column unique constraints
- **Drizzle composite primary keys**: Use `primaryKey({ columns: [table.col1, table.col2] })` in the table's second parameter function
- **Multi-user refactor approach**: This is a "one-shot migration" - schema changes define final state immediately, service refactoring follows in separate stories
- **Temporary uid placeholders**: Use constants like `SEED_UID = 'SYSTEM'` for seed/test data until services are refactored to accept uid parameters
