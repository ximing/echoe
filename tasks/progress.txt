## Note 字段模型重构 Progress

## Codebase Patterns
- Server types live in `apps/server/src/types/` directory
- DTO types live in `packages/dto/src/echoe.ts` and exported from `packages/dto/src/index.ts`
- Drizzle schema files in `apps/server/src/db/schema/` must be exported from `schema/index.ts`
- Always build (`pnpm build`) before generating migrations - drizzle-kit reads from dist/ folder
- `getDatabase()` singleton from `connection.ts` is used for all DB access
- TypeDI `@Service()` decorator on all service classes
- Note fields stored as `flds` (joined by `\x1f`), `fldNames` (JSON array), `sfld` (sort field), `csum` (checksum)
- Server lib modules live in `apps/server/src/lib/` directory
- `@tiptap/core`'s `generateHTML` requires DOM; use `jsdom@20` (CJS-compatible) as polyfill in Node.js. jsdom v28+ uses ESM and is incompatible with the current CommonJS jest config
- Jest tests use CommonJS transform (ts-jest with `module: 'commonjs'`); ESM node_modules will fail unless `transformIgnorePatterns` is updated

  - `serializeToHtml(doc: ProseMirrorJsonDoc): string` - converts ProseMirror JSON to HTML using `@tiptap/core`'s `generateHTML`

- **Learnings for future iterations:**
  - `@tiptap/core`'s `generateHTML` internally calls `document.implementation.createHTMLDocument()` via `prosemirror-model`'s DOMSerializer - it needs a real DOM even in Node.js

## 2026-03-12 - US-006
- What was implemented: Created `apps/server/src/lib/note-field-normalizer.ts` with the `normalizeNoteFields` function
  - `NormalizerInput`: `{ notetypeFields: string[]; fields?: Record<string, string>; richTextFields?: RichTextFields }`
  - `NormalizerOutput`: `{ fieldsJson: CanonicalFields; fldNames: string[]; flds: string; sfld: string; csum: number }`
  - richTextFields converted to HTML via `serializeToHtml`, plain text for sfld via `serializeToPlainText`
  - flds = field values joined by `\x1f` in notetypeFields order
  - sfld = plain text of first field (strips HTML tags for plain-text inputs, uses serializeToPlainText for richText inputs)
  - csum = `parseInt(sha1(sfld).slice(0, 8), 16)` using Node.js `crypto` module
  - richTextFields take precedence over fields for the same key
- Files changed:
  - `apps/server/src/lib/note-field-normalizer.ts` (new file)
  - `apps/server/src/__tests__/note-field-normalizer.test.ts` (new file, 12 tests)
- **Learnings for future iterations:**
  - For sfld computation: if first field came from richTextFields, use `serializeToPlainText` on the original doc; otherwise strip HTML tags with regex
  - `computeCsum` uses `createHash('sha1').update(sfld, 'utf8').digest('hex')` then `parseInt(hash.slice(0, 8), 16)`
  - normalizeNoteFields is a pure function (no DB access, no side effects) - easy to test and reuse
  - All 56 tests pass (4 test suites) after adding 12 new tests for this module
